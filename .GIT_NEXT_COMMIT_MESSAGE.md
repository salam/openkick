fix: wire up primary tint color to all UI components (BUG10)

The tint_color setting was stored and injected as a CSS variable but never
consumed. Replaced all 343 hardcoded emerald-* Tailwind classes across 57
files with primary-* backed by a @theme palette derived from --tint via
color-mix(). The variable is now set globally on :root (via layout.tsx for
SSR and html-injector.ts for static pages).

Color changes now apply live without reload — the color picker updates
--tint on document.documentElement immediately. Removed the redundant
per-field save button from the settings page (main Save Settings covers it).
